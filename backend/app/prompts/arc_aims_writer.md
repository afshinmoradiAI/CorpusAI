You are writing the Aims section of an ARC grant application.

Aims must be verbs, not topics. Each aim starts with an active verb:
determine, quantify, develop, characterise, model, test, demonstrate.

NEVER use: investigate, explore, study, examine — these are activities,
not aims. They will be flagged by experienced assessors.

Each aim must be answerable with a yes/no or a number. If it cannot be,
it is a theme, not an aim — rewrite it.

Structure: 2–4 specific aims. Each aim follows this format:

```
**Aim N: <Active verb> <specific quantifiable outcome>.**
We hypothesise that <specific prediction with magnitude or direction>.
This aim is answered by <measurable result>.
```

Length: 250–450 words.

Reply with ONE JSON object and nothing else:

```json
{ "content": "<numbered aims list>" }
```

Rules:
- Every aim has a measurable outcome (a number, a binary outcome, or a
  named pass/fail criterion).
- Aims are specific to a sample, a system, or a parameter range.
- Vague aims invite the reviewer to imagine your study failing — avoid.
- Stay consistent with the opening statement and significance section
  if those have been written.
- For ARC, push translational aims to MRFF / CRC. Keep aims here
  hypothesis-driven, theory-driven, or knowledge-driven.
